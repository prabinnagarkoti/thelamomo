BizMenu Builder — User Stories

Project: BizMenu Builder (ThelaMomo)  |  Team: Prabin, Sandip, Kulwinder  |  Total Stories: 24

Prabin — Frontend UI, Homepage, Menu Display, Cart, Checkout & Maps

# Epic User Story Acceptance Criteria Priority Status
P-01 EP-02 As a customer, I want to view the restaurant homepage with a hero section so that I get a strong first impression of the business. Hero section displays restaurant name, tagline, badges, and "Browse Menu" / "View Cart" buttons; popular items appear in a highlight card. High Done
P-02 EP-02 As a customer, I want to filter the menu by category so that I can find items easily. Category filters (All, Main Course, Starters, Desserts, Beverages) display correctly; items update without page reload. High Done
P-03 EP-02 As a customer, I want to search items by name so that I can locate specific dishes quickly. Search input filters displayed items in real time without page reload. Medium Done
P-04 EP-02 As a customer, I want to add items to a cart and adjust quantities so that I can manage my order before checkout. Cart opens as a side sheet; quantity can be increased, decreased, or item removed; total updates instantly. High Done
P-05 EP-02 As a customer, I want to proceed to checkout with my cart items so that I can place an order online. Checkout page shows order summary, contact form (name, email, phone), and delivery address field; order is submitted and saved to MongoDB. High Done
P-06 EP-02 As a customer, I want to use Google Maps autocomplete to enter my delivery address so that my location is accurate. Google Places autocomplete suggests addresses; selected address auto-fills the field; an interactive map with a draggable pin shows the location. Medium Done
P-07 EP-02 As a customer, I want to see a "Popular Right Now" section on the homepage so that I can quickly discover top dishes. First 3 available menu items are shown in a glassmorphism card on the hero section with name, description, price, and image. Low Done
P-08 EP-02 As a customer, I want to view the About, Reviews, and Contact sections on the homepage so that I can learn more about the restaurant. About section shows restaurant story and values; Reviews section displays 3 testimonial cards; Contact section shows location, hours, and phone/email. Medium Done

Sandip — Authentication, User Management, Security, Order Tracking & Messaging

# Epic User Story Acceptance Criteria Priority Status
S-01 EP-04 As a customer, I want to register an account with email and password so that I can place orders. Registration form collects name, email, password, and confirm password; password strength meter shows requirements (8+ chars, uppercase, lowercase, number, special char); account is saved to MongoDB with hashed password. High Done
S-02 EP-04 As a customer, I want to log in with my credentials so that I can access my account. Login form accepts email and password; on success, user is redirected to homepage; on failure, a clear error message is displayed with a link to register. High Done
S-03 EP-04 As a customer, I want to be prompted to sign in when I try to add items to the cart without being logged in so that only authenticated users can place orders. If not logged in, clicking "Add to Cart" opens the cart sheet with a sign-in prompt containing "Sign In" and "Register" buttons. High Done
S-04 EP-04 As a customer, I want to be redirected to sign in if I access the checkout page without logging in so that orders are always linked to an account. Checkout page shows a "Sign In Required" card with Sign In and Register buttons if user is not authenticated. High Done
S-05 EP-02 As a customer, I want to track my order status after placing it so that I know when my food will be ready. After placing an order, user is redirected to /order/[id]; the page shows order status (In Process / Delivered / Cancelled), items, total, and customer details. High Done
S-06 EP-02 As a customer, I want to send messages to the restaurant owner about my order so that I can communicate special requests or ask questions. Order detail page has a message input; messages are stored in the order document with sender type (customer/owner) and timestamp; both parties can view the conversation. Medium Done
S-07 EP-04 As a developer, I want user roles (customer/owner) to be enforced so that only owners can access the dashboard. User model has a role field (customer or owner); dashboard pages check session role; Navbar conditionally shows "Dashboard" link only for owners. High Done
S-08 EP-04 As a developer, I want restricted users to be blocked from logging in so that banned accounts cannot access the system. User model has a restricted field; if restricted is true, the login returns an error message and session is not created. Medium Done

Kulwinder — Backend, Database, Owner Dashboard, Menu Management, Deployment & Config

# Epic User Story Acceptance Criteria Priority Status
K-01 EP-03 As a developer, I want to connect the app to MongoDB Atlas so that menu data is stored persistently. Connection established via Mongoose; MenuItem, Order, User, and RestaurantConfig schemas created; data persists after restart. High Done
K-02 EP-01 As a restaurant owner, I want to add menu items with name, description, price, category, and image so that customers can see my offerings. Owner inputs name, description, price, category; item appears on customer menu immediately; image upload is supported. High Done
K-03 EP-01 As a restaurant owner, I want to edit and delete menu items so that I can keep the menu current. Modified items update instantly on the customer-facing menu; deleted items are removed from customer view. High In Progress
K-04 EP-01 As a restaurant owner, I want to view and manage all orders from a dashboard so that I can track business activity. Dashboard overview shows stat cards (Active Orders, Delivered, Cancelled, Revenue); recent orders listed with customer name, total, status, and date. High Done
K-05 EP-01 As a restaurant owner, I want to update order status (In Process → Delivered / Cancelled) so that customers are informed of progress. Order management page lists all orders; owner can change status via dropdown or buttons; status change is saved to MongoDB and reflected on customer's order tracking page. High Done
K-06 EP-01 As a restaurant owner, I want to configure restaurant details (name, tagline, about text, address, phone, hours, logo) so that the storefront reflects my brand. Settings page allows editing all RestaurantConfig fields; saved changes are reflected on the Navbar, hero section, about section, and contact section immediately. Medium Done
K-07 EP-01 As a restaurant owner, I want to receive new customer messages on the dashboard so that I can respond quickly. Dashboard "New Messages" panel shows latest unread customer messages with sender name, text preview, and timestamp; clicking opens the order detail with full conversation. Medium Done
K-08 EP-03 As a developer, I want to deploy the app on Vercel so that it is publicly accessible. App builds successfully with next build; accessible via public URL; environment variables (MongoDB URI, NextAuth secret, Google Maps key) configured on Vercel; CI/CD auto-deploys from GitHub. High In Progress
